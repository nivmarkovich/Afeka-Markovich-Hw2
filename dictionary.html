CNN & Artificial Intelligence Glossary

Exactly 40 terms with clear definitions for your university reference.

Model Dictionary
1. Artificial Intelligence (AI)
The field of computer science focused on building systems that perform tasks requiring human-like perception, reasoning, or decision-making.
2. Machine Learning (ML)
A subset of AI where models learn patterns from data instead of being programmed with explicit rules for every case.
3. Neural Network
A computational model made of interconnected units (neurons) that transform inputs through weighted sums and activation functions.
4. Convolutional Neural Network (CNN)
A neural network architecture designed for grid-like data (especially images), using convolution layers to detect local spatial patterns.
5. Neuron
A single computational unit that computes a weighted sum of its inputs, adds a bias, and passes the result through an activation function.
6. Layer
A stage in the network where a group of neurons or filters processes data and passes the result to the next stage.
7. Weights
Learnable parameters that scale each connection or filter coefficient; training adjusts weights to reduce error.
8. Bias
An extra learnable offset added after the weighted sum, allowing the model to shift activations without changing input patterns.
9. Convolution
An operation that slides a small kernel (filter) over the input, computing dot products at each position to produce a feature map.
10. Filter (Kernel)
A small matrix of weights used in convolution to detect features such as edges, corners, or textures.
11. Feature Map
The output of applying one filter across the entire input; each map highlights where a particular pattern is present.
12. Channel
A separate slice of data in a tensor (e.g., red, green, blue in color images, or one channel for grayscale).
13. Stride
The step size by which a filter moves across the input; a larger stride reduces output spatial dimensions faster.
14. Padding
Extra pixels added around the input border so convolution can preserve spatial size or control boundary behavior.
15. Pooling
A downsampling operation that reduces spatial resolution while keeping strong activations, commonly max or average pooling.
16. Max Pooling
Pooling that keeps only the maximum value in each local window, making the representation more robust to small shifts.
17. Activation Function
A nonlinear function applied after the weighted sum (e.g., ReLU) so the network can model complex relationships.
18. ReLU (Rectified Linear Unit)
An activation defined as max(0, x); it is simple, fast, and helps reduce the vanishing-gradient problem in deep networks.
19. Dense Layer (Fully Connected)
A layer where every neuron connects to all inputs from the previous layer, often used after convolution for final classification.
20. Flattening
Reshaping multi-dimensional feature maps into a one-dimensional vector so they can feed into a dense layer.
21. Forward Pass
Propagating input data through the network layer by layer to produce outputs (logits or probabilities).
22. Backpropagation
The algorithm that computes gradients of the loss with respect to each weight by applying the chain rule from output to input.
23. Loss Function
A scalar measure of how wrong the model’s predictions are; training aims to minimize this value.
24. Cross-Entropy Loss
A common classification loss that penalizes confident wrong predictions; pairs naturally with softmax outputs.
25. Softmax
A function that converts a vector of logits into a probability distribution that sums to 1 across classes.
26. Gradient Descent
An optimization method that updates weights in the direction opposite to the gradient of the loss, scaled by a learning rate.
27. Learning Rate
A hyperparameter controlling the step size of weight updates; too large causes instability, too small slows learning.
28. Epoch
One complete pass through the entire training dataset (or a full training cycle over all provided samples).
29. Batch
A subset of training examples processed together before updating weights; batching balances speed and gradient stability.
30. Training Set
Data used to fit model parameters by minimizing loss through forward pass and backpropagation.
31. Validation Set
Held-out data used during development to tune hyperparameters and monitor generalization without training on it directly.
32. Test Set
Data reserved for final evaluation after training is complete, estimating performance on unseen examples.
33. Accuracy
The fraction of predictions that match the correct class labels, often reported as a percentage.
34. Error Rate
The proportion of incorrect predictions (often 1 minus accuracy), used to summarize classification mistakes.
35. Overfitting
When a model memorizes training noise and performs poorly on new data because it is too complex or trained too long.
36. Underfitting
When a model is too simple or under-trained to capture important patterns, yielding poor performance even on training data.
37. Hyperparameter
A setting chosen before training (e.g., learning rate, number of filters) rather than learned from data like weights.
38. Grayscale Image
An image represented by single intensity values per pixel instead of separate color channels, often normalized between 0 and 1.
39. Tensor
A multi-dimensional array of numbers (e.g., height × width × channels) used to represent inputs, weights, and activations.
40. Inference (Prediction)
Using a trained model’s forward pass only—without weight updates—to classify or score new inputs.
